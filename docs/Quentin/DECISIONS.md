- **Decision (A1, validate at the boundary):** `api/routes.py::_parse_modified_at` now validates every individual value in the `modified_at` `{name: iso}` map with `datetime.fromisoformat`, not just the JSON envelope. - **Decision (A6, wiring test):** added `tests/surface/api/test_routes.py::test_ingest_wires_modified_at_into_stored_chunks` ‚Äî a multipart `POST /ingest` through `TestClient` with a real `IngestPipeline` (fakes for parser/chunker/embedder/store, not the route's default `_StubIngest`) and a `modified_at` map, asserting the **stored** `Chunk`/`Hit` carries the validated mtime, and that a file whose value fails A1 validation lands with `modified_at=None` rather than the garbage string. This was previously untested ‚Äî `_parse_modified_at` had unit coverage of the envelope but nothing proved the value actually reached a stored chunk end-to-end.
- **Why:** README ¬ßF2 + CLAUDE.md's model-pin-guard spirit ("refuse rather than silently trust bad input") extended to recency metadata: a recency signal that can be beaten by garbage, or that treats "we don't know" as ranking evidence, is worse than no signal at all, because it fails silently and confidently. All three fixes were driven by failing tests written first from the audit's exact scenarios (corrupted-vs-real, known-old-vs-unknown-new-index, equal-mtimes-tie), confirmed red against the pre-fix code, then made green.
- **Alternatives:** (a) reject the whole upload on any invalid `modified_at` entry ‚Äî rejected, over-punishes a batch for one bad timestamp when the recency hint is best-effort by design (D28). (b) clamp an invalid value to `indexed_at` instead of dropping it ‚Äî rejected, conflates "we don't have this file's mtime" with "we do, and it's now," which is exactly the false-evidence failure mode A1/A2 exist to prevent. (c) keep the two-field iteration loop in `_is_newer` and just wrap each `getattr` in a `try/except` ‚Äî rejected, doesn't express the "evidence beats no evidence" asymmetry (one valid + one missing must not silently fall through to indexed_at) as clearly as an explicit staged rule.
