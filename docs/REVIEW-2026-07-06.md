## 🔴 High severity

### 1. `/status` leaks all per-consumer bearer tokens in plaintext
**`src/sift/api/routes.py:45` · category: secrets**

`_redacted_settings` (routes.py:50–55) redacts only the keys in `_SECRET_KEYS` = `{turso_auth_token, embed_api_key, llm_api_key, ingest_token, ocr_api_key}`. The `auth_tokens` Settings field (config.py:129) holds every per-consumer bearer credential in the literal form `"name1:token1,name2:token2"` and is **not** in that set, so it is emitted verbatim in the `GET /status` response. `/status` authenticates *any* valid token, so a consumer holding just one per-consumer token can read every other consumer's plaintext token. The ingest token is redacted but the equally-powerful per-consumer tokens are not — an inconsistency that becomes a credential-disclosure hole the moment tenants diverge in privilege.

**Fix:** Add `"auth_tokens"` to `_SECRET_KEYS` so it reports as a presence flag only.

**Note:** Today all tokens resolve to the same `"default"` tenant with identical privileges, so no *extra capability* is gained yet — but it is still plaintext secret disclosure enabling impersonation, and severity rises sharply under the planned multi-tenancy.

## 🟡 Medium severity

### 2. Conversation schema re-migrated on every store job
**`src/sift/adapters/conversation/libsql.py:257` · category: redundant-recomputation**

`_ensure_schema` runs at the top of **every** executor job (append/history/prune/set_title/list/get/delete) and issues ~6 statements each time: `CREATE TABLE IF NOT EXISTS`, four `pragma_table_info` column probes, and a second `CREATE TABLE`. The chat hot path makes 4 conversation-store calls per user message (answer.py:510–512 + 627) ⇒ **~24 redundant statements per turn**, each a network round-trip against remote Turso, purely re-probing migrations that can never change after the first run. The store uses a single-worker executor over one long-lived connection, so a one-time flag is race-free.

**Fix:** Gate `_ensure_schema` behind an `if self._migrated: return` flag set after the first successful run. Fresh-process correctness is preserved (the first job still migrates). This also resolves finding #9.

### 3. Every streamed token re-parses markdown of all prior chat turns
**`web/src/Chat.tsx:656` · category: redundant-recomputation**

Each `answer_delta` SSE frame calls `patchAssistant → setTurns` (Chat.tsx:596), re-rendering the whole `turns.map(...)` block. Turns render inline with no memo boundary, and `ChatMarkdown` (react-markdown + remark-gfm) is never memoized — verified that react-markdown v10 re-parses its full input on every render. For a conversation with **T** prior turns and an answer of **N** tokens, the completed turns' markdown is re-parsed **T×N** times even though their text never changes. Only the actively-streaming turn actually changes. The non-streaming turn objects keep referential identity through `.map`, so the fix is clean.

**Fix:** Extract the assistant-turn render into a `React.memo`-wrapped subcomponent keyed on the stable `turn` object (or memoize `ChatMarkdown` output on `text`). Bounds streaming work to re-parsing only the active turn.

## 🟢 Low severity — Security

### 4. Non-timing-safe bearer token comparison
**`src/sift/api/deps.py:46` · category: authn**

`resolve_tenant` compares the presented token against the ingest secret with `==`, which short-circuits at the first differing byte — a classic timing side-channel (CWE-208). No constant-time compare exists anywhere in `src/sift/` (grep-confirmed). Largely theoretical over a network for an interpreted-string compare, and the system is a localhost-only PoC, so this is defense-in-depth hardening.

**Fix:** Use `hmac.compare_digest(token, settings.ingest_token)` (guard the empty case), and route the per-consumer token check through a constant-time compare rather than dict hashing.

### 5. Agent bearer token persisted in world-readable cleartext
**`agent/config.py:99` · category: secrets-management**

`save()` writes the engine bearer token to `config.json` as plaintext via `open(path, "w")`, and `config_path()` creates the dir with `os.makedirs(exist_ok=True)` and no mode. Both inherit the process umask (typically 022) ⇒ 0o755 dir / 0o644 file, readable by every local user. The token authorizes `/ingest`, `/documents`, and `DELETE /documents/{hash}`. Partially mitigated in practice because the parent XDG/home dirs are usually 0700.

**Fix:** Create the dir `0o700` and write the file `0o600` (`os.open` with mode, or `os.chmod` after write); ideally store the token in the OS keyring.

## 🟢 Low severity — Efficiency

### 6. `_collapse_versions` recomputes shingles O(n²) per search
**`src/sift/pipelines/search.py:125`**

The greedy version-collapse scan (default on) is O(n²) pairwise, and `_lexical_similarity` rebuilds *both* shingle sets from scratch on every call. With `retrieve_k=30` all-distinct candidates that is ~870 shingle-set constructions vs. 30 if each were computed once. Runs synchronously inside the async `search()` coroutine. Bounded by `retrieve_k=30`, so absolute cost is small — a legitimate micro-optimization, not a scaling hazard.

**Fix:** Precompute each hit's shingle set once before the loop, pass prebuilt sets to a Jaccard helper — O(n²) → O(n) `_shingles` calls.

### 7. `list_documents` loads whole corpus then paginates in Python
**`src/sift/pipelines/tools.py:215`**

`_list_documents_executor` fetches every matching document then slices `[offset:offset+limit]` in Python; the underlying SQL has no `LIMIT`/`OFFSET`. Every call (including per-request agent calls) materializes and deserializes all **N** tenant rows regardless of page size.

**Fix:** Push `LIMIT`/`OFFSET` into the query (extend the `list_documents` port) and return a separate `COUNT` for the total.

### 8. `TokenChunker.chunk` runs blocking tokenization on the event loop
**`src/sift/adapters/chunking/token.py:119`**

`chunk()` is `async` but never awaits — it does a full `encode()` of the document plus a `decode()` of every sliding window synchronously on the event-loop thread, driven by the async `/ingest` endpoint. Its sibling `MarkitdownParser` already offloads identical work via `asyncio.to_thread` (markitdown.py:143), establishing the pattern the chunker doesn't follow.

**Fix:** Move the encode/window/decode body into a sync helper and `return await asyncio.to_thread(self._chunk_sync, doc)`.

### 9. `_get_conversation_job` runs schema migration twice per call
**`src/sift/adapters/conversation/libsql.py:371`**

Calls `_ensure_schema` at line 369, then `_history_job` at 371 which calls `_ensure_schema` again (line 314) — doubling the ~6 schema statements for a single `GET /conversations/{id}`. Naturally fixed by the one-time flag in finding #2, or by inlining the history read.

### 10. `prune_expired` issues per-conversation COUNT+DELETE (N+1)
**`src/sift/adapters/conversation/libsql.py:336`**

After selecting stale ids, the loop runs three statements per stale conversation (`_COUNT_TURNS`, `_DELETE_CONVERSATION`, `_DELETE_CONVERSATION_META`) ⇒ `1 + 3M` round-trips. `prune_expired` runs on every chat message, so a backlog scales per-message latency linearly. The `_COUNT_TURNS` result is even discarded by the caller — pure waste. Steady state returns zero rows, so this only bites after downtime or a TTL change.

**Fix:** Collapse to set-based DELETEs keyed on the cutoff (`WHERE conversation_id IN (SELECT ... HAVING MAX(created_at) < ?)`); drop the unused count.

## 🟢 Low severity — Dead code

### 11. `CrossEncoderReranker` stores `self._model`, never reads it
**`src/sift/adapters/rerank/crossencoder_http.py:24`**

The constructor assigns `self._model = model` but nothing ever reads it; the `/rerank` payload is built from `query` + `texts` only. `factory.py:245` passes `settings.rerank_model` in, so a configured value is silently discarded. (Benign — TEI's single-model `/rerank` doesn't need it.)

**Fix:** Either include the model in the payload, or drop the `model` param and the attribute (and stop passing it at factory.py:245).

### 12. Unused constant `_CONVERSATIONS_TABLE_EXISTS`
**`src/sift/adapters/conversation/libsql.py:90`**

Defined at lines 90–92 as a `sqlite_master` existence probe, never referenced anywhere (grep returns only the definition). The store runs `_ensure_schema()` on every job instead, so the guard is never needed — leftover cruft.

**Fix:** Delete lines 90–92.
